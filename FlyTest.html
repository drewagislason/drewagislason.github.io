A C Test Framework/API for testing C functions and objects

This is a simple but powerful test framework for C programs. Allows any C module to be tested independant of the overall system, thereby reducing bugs in the final code.

Features:

  1. Easily run manual tests, automated tests or both
  2. Run "smoke" or quick tests only for rapid verification of new code for CI/CD
  3. Verbosity flags for better context/understanding of the bug
  4. Filter which tests are run by keyword
  5. Automatically creates formal test logs for records / searching
  6. Test case, test group, test suite framework
  7. TestPrintf() can aid debugging and is compiled out of production code
  8. Recordes time consumed by test cases and suites
  9. And more

The test framework generally is built to test modules (that is, a C collection of functions or a C "object"). This is comprised of 1 or more test cases.

Usage = TestSuite [test_suite_args] [-a] [-f filter] [-m] [-v]

Nomenclature. See also https://www.istqb.org

  • test case - the basic unit of test
  • test suite - a set of test cases to test a module or the entire program
  • unit-test - a test suite that tests functions/behavior not exposed via API
  • smoke test - quick test cases used for CI/CD
  • fuzz tests - throws bad input at functions to verify how they handle invalid inputs
